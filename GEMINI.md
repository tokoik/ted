# プロジェクト開発方針と環境定義 (GEMINI.md)

本ドキュメントは、`ted-openxr` プロジェクトにおけるビルド構成、開発・動作環境に関する要件を定義します。

## 1. 開発環境・環境設定

- **統合開発環境 (IDE)**: Visual Studio 2022 以降
- **ビルドシステム**: CMake 3.22 以降
- **開発言語**: C++17 (`/std:c++17`)
- **ターゲットアーキテクチャ**: 64bit (x64)
- **プラットフォーム**: Windows (macOS や Linux は対象外)
- **ファイル文字コード**:
  - **C++ ソースファイル（`.h`, `.cpp`）**: 文字コードは `UTF-8` とし、**BOM を付与**します。
  - **GLSL ソースファイル（`.vert`, `.frag`, `.geom`, `.comp`）**: 文字コードは `UTF-8` とし、**BOM は付与しません**。
  - **Markdown ファイル（`.md`）**: 文字コードは `UTF-8` とし、**BOM は付与しません**。

## 2. Visual Studio フィルタ構成定義

- **Source Files**: C++ のソースファイル (`.cpp`)
- **Header Files**: C++ のヘッダファイル (`.h`)
- **Shader Files**: GLSL のソースファイル (`.vert`, `.frag`, `.geom`, `.comp`)
- **ImGui Files**: Dear ImGui および Native File Dialog Extended のソース・ヘッダファイル
- **フォルダ直下**: `config.json` は CMakeLists.txt と同レベルの `ted` フォルダ直下に配置する。その他の JSON/画像/動画/3Dモデルファイルはソリューションエクスプローラーには表示しない。

## 3. 使用する主要技術・外部ライブラリ

- **グラフィックス**: OpenGL 4.3 以降（GLSL 4.3 以降）
- **ウィンドウ・入力**: GLFW 3.4
- **UI**: Dear ImGui 1.92.8 / Native File Dialog Extended 1.3.0
- **メニューフォント**: NotoSansCJKjp-Regular.otf
- **JSONパース**: picojson
- **画像処理**: OpenCV 4.13.0
- **VR**: OpenXR SDK 1.1.61
- **ハンドトラッキング / 外部デバイス**:
  - Leap Motion SDK 4.1.0 (Orion)
  - OvrvisionPro SDK 2.0
- **その他**: GStreamer は使用しない。

## 4. 外部ライブラリの自動管理化 (libs)

- プロジェクトトップの `libs` ディレクトリ以下に、CMake 実行時に必要なライブラリやヘッダ、フォントを自動ダウンロード・配置する構成とする（ジャンクションの廃止）。
- ビルド先のバイナリディレクトリには、実行に必要な DLL、シェーダーファイル、リソースファイルを自動的にコピーする。

## 5. アーキテクチャと責務の分離

- UI クラスは画面の構築とユーザー入力の収集を担当し、デバイスの起動・停止、描画資源の再構築、投影行列の更新などの副作用を直接実装しない。
- 副作用を伴う操作は、対象を所有するクラスの操作 API に集約する。UI は API に要求を渡し、成否を受け取るだけにする。
- 必須の依存オブジェクトはポインタではなく参照で受け取り、`nullptr` を有効な状態として扱わない。
- クラスの状態は原則として非公開とし、別クラスから公開フィールドを直接編集しない。状態変更に検証や追加処理が必要な場合は、必ずその処理を含む API を設ける。
- UI や上位層へ、Media Foundation の `GUID` のようなバックエンド固有型を公開しない。境界ではバックエンド非依存の DTO またはインターフェースへ変換する。
- グローバルな `defaults` と `attitude` への依存は新規コードで増やさない。既存依存は、設定値や操作インターフェースを明示的に渡す形へ段階的に置換する。
- `GgApp` から TED 固有の `Config`、`Attitude`、`Scene`、`Camera` 依存を将来的に分離し、GLFW・OpenGL・OpenXR を扱う汎用ウィンドウ層と TED 本体を分ける。

## 6. 操作 API の設計方針

- 表示モードの変更は `GgApp::Window::setDisplayMode()` を使用する。この API が OpenXR の開始・停止、Quad Buffer の利用可否判定、設定値の更新、ビューポートの更新を一括して行う。
- クリップ面の変更は `GgApp::Window::setClipPlanes()` を使用する。この API が値の妥当性を確認し、設定値と投影行列を更新する。
- ミラー、シーン、メニューの表示状態は `is*Visible()`／`set*Visible()` を介して操作する。
- Leap Motion の有効・無効は `GgApp::setLeapMotionEnabled()` を使用し、`Menu` などから `Scene::startLeapMotion()`／`stopLeapMotion()` を直接呼ばない。
- 姿勢調整 UI は `Attitude` の取得・変更 API を使用する。ImGui の入力領域がポインタを要求する場合は一時値を編集し、変更が確定した時点で setter に渡す。
- カメラ選択 UI は `CameraCapabilities` を使用する。`CamMf::VideoFormat` や Media Foundation の型を直接参照しない。

## 7. 設定変更と資源管理

- 設定ファイルの再読み込みでは、現在使用中の `defaults` を直接上書きしない。現在値から作成した候補 `Config` に読み込み、候補から必要な描画資源を構築する。
- 候補設定と新しい描画資源の検証に成功した場合だけ、現在の設定と資源を確定する。失敗時は候補を破棄し、実行中の設定と資源を維持する。
- 外部ライブラリが確保したメモリは、そのライブラリが指定する解放関数で必ず解放する。Native File Dialog が返すパスは、コピー後に `NFD_FreePath()` で解放する。
- キャッシュは高コストな列挙や変換を避ける目的に限定し、キャッシュの無効化条件をデバイス番号やコーデックなどで明示する。

## 8. 変更時の確認

- 変更後は最低限 `git diff --check` を実行する。
- ビルド構成が利用可能な場合は Debug または Release の少なくとも一方をビルドする。
- 外部依存の欠落などによりビルドできない場合は、失敗理由と未確認範囲を作業記録に残す。
- 設定再読み込み、デバイス切り替え、OpenXR の開始・停止のような失敗し得る処理では、成功時と失敗時の両方について状態遷移を確認する。
